# kibana-fields-generator

`kibana-fields-generator` is a Go utility that automatically generates strongly-typed Go field definitions, validation helpers, and test suites from the official [Elastic Common Schema (ECS)](https://www.elastic.co/guide/en/ecs/current/ecs-field-reference.html) specification (`ecs_nested.yml`).

The output generated by this repository powers the [`github.com/iostrovok/kibana-fields`](https://github.com/iostrovok/kibana-fields) library.

## Features

- **Automated Schema Fetching**: Downloads the ECS YAML definition for a given version tag directly from Elastic's official repository.
- **Strongly Typed Fields**: Generates Go packages and type definitions for each ECS field set (`agent`, `process`, `network`, `http`, etc.).
- **Validation & Check Helpers**: Generates check and validation functions for verifying field values and types.
- **Automated Test Suite**: Output includes unit tests to ensure syntax and schema correctness.

## Requirements

- Go 1.21+

## Usage

Run the generator by passing the destination output path and optional ECS version tag:

```bash
go run ./main.go --version=<ECS_VERSION> --path=<OUTPUT_DIRECTORY>
```

### Command Line Flags

| Flag | Default | Description |
| --- | --- | --- |
| `--path` | `""` | *(Required)* Target directory where generated files should be saved. |
| `--version` | `"9.4"` | Elastic Common Schema version tag to fetch (e.g. `8.16`, `9.4`). |

### Example

Generate ECS version 8.16 field definitions into a `./result` directory:

```bash
go run ./main.go --version=8.16 --path=./result
```

## Makefile Targets

- `make run` — Runs the generator with default parameters (`--version=8.16 --path=./result`).
- `make test` — Runs unit tests.
- `make mod` — Tidies and vendors Go dependencies (`go mod tidy`, `go mod vendor`).

## Project Structure

- `main.go` — Entry point for CLI flags parsing and orchestration.
- `templates/` — Templates used to build output Go files, tests, checks, and documentation.
- `internals/` — Core packages for downloading, parsing YAML schema, and writing generated code:
  - `internals/yaml/` — Fetches and parses `ecs_nested.yml`.
  - `internals/set/` — Code generation logic for field sets, check files, and tests.
  - `internals/face/` — Data structures representing ECS schema elements.
